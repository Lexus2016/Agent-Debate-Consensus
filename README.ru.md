## Зачем Agent Debate Consensus?

Один AI даёт вам одну точку зрения. Agent Debate Consensus даёт четыре — одновременно, в реальном времени, с аргументами друг против друга.

Разные модели обучены на разных данных, имеют разные предубеждения и разные стили рассуждения. Заставляя их дебатировать на тему и защищать свои позиции перед оппонентами, вы получаете более полную картину, чем способна дать любая отдельная модель. Противоречия выходят на поверхность. Слабые аргументы оспариваются. Консенсус, когда он достигается, — заработан.

Используйте платформу, чтобы проверить идею на прочность, изучить спорную тему с нескольких углов или наблюдать, как передовые модели подталкивают друг друга к более точным ответам.

## Возможности

- **Многоагентные дебаты**: До 4+ AI-агентов (Kimi K2, Gemini 3 Pro, Claude Haiku 4.5, Grok 4.1 Fast) участвуют в структурированных дебатах одновременно
- **Стриминг в реальном времени**: Потоковая передача ответов токен за токеном через Server-Sent Events — без ожидания полного ответа
- **Система @упоминаний**: Адресуйте вопросы конкретным агентам с помощью `@Kimi`, `@Gemini`, `@Claude`, `@Grok`
- **Роль модератора**: Вмешивайтесь в любой момент, чтобы перенаправить дебаты, добавить контекст или подтолкнуть агентов к решению
- **Умный движок ответов**: Очередь с приоритетами и задержками не позволяет агентам «говорить» одновременно
- **Каталог агентов**: Просматривайте и добавляйте 400+ моделей из каталога OpenRouter
- **Расширенное мышление**: Полная поддержка моделей с возможностями рассуждения и chain-of-thought
- **Экспорт в Markdown**: Скачайте всю дискуссию в формате `.md` для документации или передачи коллегам
- **Копирование сообщений**: Скопируйте любое сообщение в буфер обмена в формате Markdown
- **Сохранение сессий**: Разговоры сохраняются в localStorage — продолжайте с того места, где остановились
- **Тёмная тема**: Современный тёмный интерфейс премиум-класса, созданный для длительных сессий

## Как это работает

1. **Постановка темы** — Вы задаёте тему или вопрос. Все активные агенты получают её в качестве контекста.
2. **Структурированные ответы** — Каждый агент формулирует независимую позицию, не зная, что готовят другие.
3. **Перекрёстный обмен** — В последующих раундах каждый агент получает предыдущие ответы других агентов, что обеспечивает настоящую реакцию и контраргументы.
4. **Модерация** — Вы вмешиваетесь в любой момент: уточняете термины, оспариваете слабый аргумент или просите конкретного агента защитить свою позицию.
5. **Сближение позиций** — Через итеративный обмен агенты уточняют позиции, признают более сильные аргументы и движутся к консенсусу — или выявляют непримиримые разногласия, о которых стоит знать.

## Установка

**1. Клонирование репозитория**

```bash
git clone https://github.com/Lexus2016/Agent-Debate-Consensus.git
cd Agent-Debate-Consensus
```

**2. Установка зависимостей**

```bash
npm install
```

**3. Настройка окружения**

Создайте файл `.env` в корне проекта:

```env
OPENROUTER_API_KEY=your_key_here
```

Получите API-ключ на [openrouter.ai/keys](https://openrouter.ai/keys).

**4. Запуск сервера разработки**

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## Использование

1. Выберите AI-агентов на боковой панели
2. Введите тему или вопрос, чтобы начать дебаты
3. Наблюдайте, как агенты обсуждают, спорят и развивают мысли друг друга
4. Используйте `@упоминания`, чтобы задать вопрос конкретному агенту
5. Вмешивайтесь в любой момент, чтобы направить обсуждение или добавить свою точку зрения
6. Агенты работают к консенсусу через структурированный диалог
